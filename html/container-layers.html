🧠 How Layers work in Kubernetes

Container images consist of read-only layers. When you run a Pod, Kubernetes pulls these layers and stacks them via Union Filesystem (OverlayFS). A thin writable container layer sits on top.

FROM ubuntu:22.04 ← Layer 1 (base)
RUN apt-get update ← Layer 2 (deps)
COPY app.jar /app/ ← Layer 3 (app)
CMD ["java","-jar"] ← metadata

🔁 Reusability: Layers are cached on each node. Multiple Pods reuse the same layers, saving disk & bandwidth.

⚙️ In a Kubernetes Node:

  • Shared layers: 100 Pods using same image → layers loaded once.
  • Copy-on-Write (COW): When a container modifies a file, it copies to its writable layer.
  • Image pull policy: If layer exists locally, it's reused.
  • InitContainers & sidecars can share volumes, but layer isolation remains.
